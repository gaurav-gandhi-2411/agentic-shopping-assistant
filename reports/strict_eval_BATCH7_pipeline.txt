Strict gold-set expansion (Batch 7) — statistical robustness pass
Run: 2026-07-11, commit ed57ebc (parent of the commit that lands this report + Batch 7 labels)
Command: python scripts/eval_strict.py --mode pipeline

WHY: Part 2 of the best-in-class-product review asked to "extend the hand-labeled gold
set for statistical robustness where it's currently thin." The two thinnest categories
were literal (n=5 queries) and multi_attribute (n=8 queries) — both well under the ~15-20
query floor needed for a single-query miss to not swing the category percentage by >10pp.

CHANGE: added 8 literal queries (5->13) and 6 multi_attribute queries (8->14, 2 of which
return EMPTY result sets — see below) to eval/fixtures/strict_gold_queries.yaml. Hand-
labeled all newly-retrieved top-5 items against the identical rubric documented in that
file's header (garment type, silhouette/style, gender, colour-family, occasion register,
budget) — eval/fixtures/strict_gold_labels.yaml "Batch 7", 90 new item labels.

BEFORE (pre-Batch-7, n=40 queries, 186 labeled items, commit 5698a99):
  overall precision@5: 0.860  (160/186)
    literal          0.960  (24/25)
    multi_attribute   0.657  (23/35)
    occasion         0.836  (56/67)

AFTER (post-Batch-7, n=54 queries, 246 labeled items):
  overall precision@5: 0.841  (207/246)
    literal          0.908  (59/65)
    multi_attribute   0.636  (35/55)
    occasion         0.836  (56/67)
    style            0.966  (57/59)

The overall number moved 0.860 -> 0.841. This is NOT a regression — zero retrieval or
ranking code changed between these two runs (retrieval snapshot commit unchanged). It is
the expected, honest effect of a larger, more representative literal/multi_attribute
sample: the original 5-query literal category was disproportionately easy (plain "colour
+ garment" queries with no attribute-stacking); the 8 new queries include budget caps,
plural-pack items, and a title/desc-contradiction case, which is a more realistic query
mix and drags the category mean down from an artificially high 0.960 to 0.908 — both are
real numbers, the larger n is simply less noise-prone (per the "n<50 treat deltas under
5-10pp as noise" convention: literal moved from n=25 items, which is well below that
floor, to n=65).
